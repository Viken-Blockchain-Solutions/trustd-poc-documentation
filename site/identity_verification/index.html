Decentralized Identity Verification Feature Documentation

Feature Overview

The decentralized identity verification feature is a groundbreaking addition to the TrustD Proof of Concept (POC) platform. This feature leverages Self-Sovereign Identity (SSI) principles to enable users to control and share their identity information securely. By employing a decentralized model, TrustD POC facilitates direct interactions among issuers, holders, and verifiers within the Transport and Logistics industry, streamlining identity verification processes. This approach enhances privacy, reduces reliance on central authorities, and increases the efficiency of verifying credentials such as Vehicle Insurance Certificates.

Technical Specifications

Architecture

The decentralized identity verification feature is built on the Dock.io platform, utilizing blockchain technology to ensure the integrity and security of verifiable credentials (VCs). The technical architecture comprises several key components:

  • Blockchain Protocol: Utilizes the Dock blockchain to record transactions and manage decentralized identifiers (DIDs).
  • Smart Contracts: Govern the issuance, revocation, and verification of VCs. These are deployed on the Dock blockchain and interact with the application logic.
  • Data Flow: The process involves three main entities: Issuers, Holders, and Verifiers.
    1. Issuers generate VCs and send them to Holders.
    2. Holders store these credentials in their Dock.io wallet app.
    3. Verifiers request and verify credentials from Holders.

User Guide

Prerequisites

  • Users must have a Dock.io wallet app installed.
  • Users should be registered on the TrustD POC platform and verified to become either an Issuer or Holder.

Verification Process

  1. For Holders:

    • Receive a VC from an Issuer.
    • Accept and store the VC in the Dock.io wallet app.

  2. For Verifiers:

    • Log into the TrustD POC verifier portal.
    • Submit a verification request for a Holder's credential.
    • Review the verification result, which includes a link to the credential and its associated document.

Troubleshooting Common Issues

  • If unable to receive or store VCs, ensure the Dock.io wallet app is updated to the latest version.
  • If verification requests fail, verify the Holder's DID is correct and the credential has not expired or been revoked.

API Documentation

Endpoints

  • Issue Credential: /api/issue-credential

    • Request Parameters: Issuer DID, Holder DID, Credential metadata
    • Response: Credential ID, issuance status

  • Verify Credential: /api/verify-credential

    • Request Parameters: Verifier DID, Credential ID
    • Response: Verification status, Credential details

Example Requests and Responses

  • Issue Credential Request json { "issuerDID": "did:dock:xyz123", "holderDID": "did:dock:abc456", "metadata": { "credentialName": "Vehicle Insurance Certificate", // Additional metadata fields } }

  • Issue Credential Response json { "credentialID": "12345", "status": "Issued" }

  • Verify Credential Request json { "verifierDID": "did:dock:uvw789", "credentialID": "12345" }

  • Verify Credential Response json { "status": "Verified", "credentialDetails": { "issuerDID": "did:dock:xyz123", "holderDID": "did:dock:abc456", // Additional credential details } }

Security Considerations

The decentralized identity verification feature incorporates multiple layers of security to protect user identities and data:

  • Blockchain Security: Utilizes the inherent security features of blockchain technology, such as immutability and encryption, to protect the integrity of credentials.
  • Data Encryption: Ensures that personal data is encrypted both in transit and at rest.
  • Selective Disclosure: Allows Holders to control which pieces of their identity information are shared with Verifiers, enhancing privacy.
